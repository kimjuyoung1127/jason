# 이 파일은 루트 작업 규칙과 아카이브/RAG 운영 기준을 정의합니다.

## 공통 우선순위
1. 작업 시작 전 현재 폴더의 `claude.md`를 먼저 읽는다.
2. 하위 폴더에 `claude.md`가 없으면 상위 폴더 규칙을 상속한다.
3. 코드를 새로 만들거나 수정할 때는 관련 파일을 최소 1개 이상 확인한 뒤 진행한다.
4. 적용 가능한 스킬을 항상 우선 참조한다. 적합한 스킬이 없으면 구현 전에 사용자에게 추가 요청한다.

## 출력 규칙
- 모든 코드 파일의 맨 위에 해당 파일 목적을 1~2문장으로 짧게 쓴다.
- 보라색 계열은 사용하지 않는다.
- 프론트엔드와 백엔드는 폴더명이 1:1로 매핑되도록 유지한다.

## 디자인 시스템
- 컬러: `#060606` bg, `#e52a2a` accent, `#ffffff` text. 보라색 절대 금지.
- 폰트: Syne(display, 800) + JetBrains Mono(body). 변경 금지.
- 모든 컴포넌트는 CSS Module 사용 (인라인 스타일, Tailwind 금지).
- 프로젝트 데이터 단일 소스: `apps/frontend/lib/projects.ts`.

## 기술 규칙
- Three.js/R3F 코드는 반드시 `dynamic(..., { ssr: false })`로 임포트한다.
- 연속 애니메이션은 `prefers-reduced-motion` 미디어 쿼리를 반드시 존중한다.
- 3D 스파이럴이 유일한 hero flourish — 다른 곳에 WebGL 씬 추가 금지.

## 아카이브 규칙 (RAG 준비)
- 모든 코딩 작업은 `archive/sessions/*.md`에 세션 단위로 기록한다.
- 검색/임베딩용 요약 로그는 `archive/index.jsonl`에 append-only로 누적한다.
- 기록 단위는 최소: 날짜, 변경 파일, 의사결정, 다음 액션.
