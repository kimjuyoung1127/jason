# 이 파일은 프론트엔드 전체 구현 규칙을 정의합니다.

## 디자인 톤
- `referecnce.md`의 톤(블랙 기반, 강한 타이포, 노이즈/질감)을 유지한다.
- 메인 히어로는 R3F 기반 3D 스파이럴 구조를 정식으로 사용한다 — CSS-only 버전으로 회귀 금지.
- 디테일 오버레이(슬라이드-인)가 랜딩 내 1차 상호작용이며, `/projects/[slug]`는 보조 SEO 루트이다.

## 반응형
- 모바일 브레이크포인트: 360, 768, 1024, 1440.
- WebGL 미지원 시 2D 카드 그리드 폴백 필수.

## 코드 규칙
- `@/*` 경로 별칭을 모든 import에 사용한다.
- 모든 R3F/Three.js 코드는 `dynamic(..., { ssr: false })`로 동적 임포트한다.
- 새 컴포넌트는 반드시 CSS Module 파일과 함께 생성한다.
- 모든 파일 첫 줄에 해당 파일의 기능 설명 주석을 작성한다.

## 컴포넌트 분류
- `components/hero/` — 3D 스파이럴 히어로 관련
- `components/ui/` — 공통 재사용 UI
- `components/sections/` — 랜딩페이지 전환 섹션
- `components/layout/` — 헤더, 레이아웃 컴포넌트
- `lib/` — 유틸리티 훅, 데이터, 헬퍼
